# ReceiTech 
Plataforma Colaborativa de Compartilhamento de Receitas**

## SPRINT 1 – INVESTIGAÇÃO DA TEMÁTICA CENTRAL ## 

### 1. Visão do Projeto ### 

**Problema:**   Muitas pessoas têm dificuldade em encontrar receitas confiáveis, organizadas e adaptadas às suas necessidades. Além disso, diversas plataformas apresentam excesso de publicidade, interfaces complexas ou pouca interação entre os usuários. 

**Solução:** O objetivo do projeto é criar uma plataforma de receitas acessível e intuitiva para todas as idades, onde qualquer pessoa possa compartilhar, descobrir e se inspirar com receitas de forma simples e agradável. 

**Visão do Projeto:**   Desenvolver uma plataforma web colaborativa de compartilhamento de receitas culinárias, permitindo que usuários de todas as idades possam publicar, pesquisar, organizar e interagir com receitas de forma simples, intuitiva e acessível. A solução busca criar uma comunidade digital onde cozinheiros iniciantes e experientes possam trocar experiências, descobrir novos pratos e preservar conhecimentos culinários por meio de uma interface amigável e recursos de interação social.

### 3. Product Charter ###
**Título do Projeto**  
ReceiTech – Plataforma Colaborativa de Compartilhamento de Receitas

**Patrocinador**   
Professor da disciplina de Gestão de Projetos – IFSP

**Gerente do Projeto**  
Livia Mendes

**Objetivo**  
Desenvolver uma plataforma web para compartilhamento e descoberta de receitas culinárias.

**Justificativa**  
Facilitar o acesso a receitas organizadas e promover a interação entre usuários interessados em culinária.

**Descrição**  
Sistema web que permitirá cadastro de usuários, publicação de receitas, busca, curtidas, comentários e favoritos.

**Premissas**   
- Disponibilidade de internet
- Uso de GitHub para versionamento
- Utilização de banco de dados
- Aplicação da metodologia Scrum



**Restrições**   
- Prazo acadêmico definido pela disciplina
- Recursos financeiros limitados
- Equipe composta apenas pelos integrantes do grupo


**Riscos**    
- Atrasos no desenvolvimento
- Problemas de integração
- Alterações nos requisitos
- Falhas técnicas

  
**Marcos**   
- Aprovação da visão do projeto
- Levantamento de requisitos
- Modelagem do sistema
- Desenvolvimento das funcionalidades

  
**Testes**  
- Apresentação final


**Orçamento**  
O projeto será desenvolvido utilizando ferramentas gratuitas e recursos acadêmicos, não havendo previsão de custos financeiros diretos durante sua execução.
